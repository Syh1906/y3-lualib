# 证据规则

使用这个 skill 时，回答必须基于当前本地仓库或地图项目中的真实文件。

## 证据优先级

1. 任务相关源码。
2. `init.lua` 挂载关系。
3. `doc/API` 文档。
4. `.luarc.json` 和 LuaLS 插件。
5. `meta/` 类型声明。
6. `README.md` 和演示代码。

演示代码只能作为样例，不作为内核规则。

## 回答格式

解释 Y3 内核时，尽量包含：

- 涉及的 `y3.xxx` 入口。
- 对应源码文件。
- 对应 API 文档。
- 同步/本地边界说明。
- 未验证的假设。

## 需要停止的情况

遇到以下情况停止并说明：

- 找不到 Y3 根目录。
- 找不到用户提到的 API 或模块。
- `doc/API` 和源码明显不一致。
- 需要判断同步安全但没有读相关源码。
- 用户要求添加兜底逻辑，但没有明确确认。

## 搜索建议

- 精确文本优先用 `rg --fixed-strings`。
- 查结构优先从 `init.lua` 和对应目录进入。
- 查 API 文档先用 `doc/API.md`。
- 查 LuaLS 行为先读 `.luarc.json`。
