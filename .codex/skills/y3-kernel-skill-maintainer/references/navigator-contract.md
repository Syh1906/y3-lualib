# Navigator 契约

`y3-kernel-navigator` 是消费侧 skill，可以进入地图项目。维护时必须保护这个边界。

## 允许内容

- 用户需求涉及的 Y3 内核能力识别。
- 基于源码证据的 Y3 API 入口建议。
- Y3 能力与地图项目扩展工具库、业务封装或公共模块的衔接边界。
- Y3 根目录定位。
- `init.lua` 加载顺序。
- 内核模块分层。
- 类系统、热重载、沙盒。
- 事件、触发器、对象事件。
- 引擎 API 桥接。
- 对象包装模型。
- 物编数据入口。
- UI 内核。
- 同步/本地边界。
- API 文档定位。
- 证据优先级。

## 边界外内容

- 开发仓同步门。
- 发布流程。
- 索引重建要求。
- maintainer skill 路径。
- 忽略地图项目已有扩展工具库、业务封装或公共模块。
- 把某一种地图项目架构写成通用规则。
- 自动兜底逻辑。
- 对普通地图项目无意义的仓库维护要求。

## 隔离检查

维护后检查：

1. `navigator/SKILL.md` 是否提到 `y3-kernel-skill-maintainer`。
2. `navigator/references` 是否提到“同步门”“发布前”“维护索引”。
3. `navigator` 是否要求普通地图项目运行维护脚本。
4. `navigator` 是否把 `演示/` 写成内核规则。

以上任一命中，都要回到 navigator 的内核导航职责重新收敛。
