# Navigator 契约

`y3-kernel-navigator` 是消费侧 skill，可以进入地图项目。维护时必须保护这个边界。

## 允许内容

- Y3 根目录定位。
- `init.lua` 加载顺序。
- 内核模块分层。
- 类系统、热重载、沙盒。
- 事件、触发器、对象事件。
- 引擎 API 桥接。
- 对象包装模型。
- 物编数据入口。
- UI 内核。
- 同步/本地边界。
- API 文档定位。
- 证据优先级。

## 禁止内容

- 开发仓同步门。
- 发布流程。
- 索引重建要求。
- maintainer skill 路径。
- 防守图、练功房、合成、商店、任务系统等玩法模板。
- 自动兜底逻辑。
- 对普通地图项目无意义的仓库维护要求。

## 隔离检查

维护后检查：

1. `navigator/SKILL.md` 是否提到 `y3-kernel-skill-maintainer`。
2. `navigator/references` 是否提到“同步门”“发布前”“维护索引”。
3. `navigator` 是否要求普通地图项目运行维护脚本。
4. `navigator` 是否把 `演示/` 写成内核规则。

以上任一命中，都要修正。
