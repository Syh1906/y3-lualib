# 更新日志

> Parent: [index.md](index.md)

本文说明 `更新日志.md` 的维护方式。

## 轨道

`更新日志.md` 使用两个轨道：

- 上游同步：来自上游基准的接口、事件、行为或兼容变化。
- 本地迭代：当前分支自己的修复、文档、工具和运行时变化。

## 平时开发

用户可见变化写入“未发布”。

```markdown
## 未发布

### 上游同步

- 暂无。

### 本地迭代

- 新增或修复内容。

### 兼容影响

- 用户升级时需要注意的影响。
```

## 本地发布

发布时把“未发布”内容归档到当前本地版本：

```markdown
## moy-2026.07.03.1 - 2026-07-03

### 上游基准

- 基于上游版本号 `260616` 和基准提交。

### 本地迭代

- 本地版本内容。
```

## 历史记录

旧日期记录保留在“历史记录”下。后续新增内容不再写入旧格式。
